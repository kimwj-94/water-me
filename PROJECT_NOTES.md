# 💧물 줘! 배포 메모

이 문서는 배포본에 포함해도 되는 중립적인 프로젝트 메모입니다.

## 목적

`💧물 줘!`는 개인이 키우는 화분의 물주기 일정을 관리하고, 구글 캘린더에 중앙 데이터와 알림용 일정을 저장하는 정적 웹앱입니다.

## 주요 설계

- 서버 없이 `index.html`, `style.css`, `app.js`, `plants-db.js`만으로 실행합니다.
- 구글 미연동 상태에서는 식물 데이터가 브라우저 `localStorage`에 저장됩니다.
- 구글 캘린더 연동 후에는 캘린더 안의 앱 데이터 이벤트가 원본이 됩니다.
- 앱이 중앙 데이터를 저장한 뒤 식물별 미래 물주기 알림 일정을 캘린더에 생성합니다.
- 저장, 수정, 삭제, 전체 재생성 작업은 내부 큐로 직렬화해 같은 화면 안에서 동기화 작업이 겹치지 않게 합니다.
- 구글 API 호출 중 토큰 만료가 감지되면 조용히 토큰을 1회 갱신하고 같은 요청을 다시 시도합니다.
- 앱은 기본적으로 1인 관리용이며, 다른 사람에게는 구글 캘린더를 공유하거나 내보내서 조회용으로 제공하는 방식을 권장합니다.
- 구글 API 설정은 별도 탭이 아니라 설정 모달에서 처음 한 번 저장하고, 이후에는 상단 구글 로그인 버튼으로 연결합니다.
- 기본 식물 도감은 300종까지 제공합니다. 보조 도감 항목은 보수적인 기본 물주기값을 제공하고, 실제 등록 시 사용자가 조정할 수 있게 합니다.

## 앱 출시 고려사항

- 현재 구조는 백엔드 서버가 없어도 PC/모바일 웹에서 같은 구글 캘린더 중앙 데이터를 사용할 수 있습니다.
- 추후 앱으로 배포할 경우 Capacitor, Trusted Web Activity, WebView 래핑 같은 방식으로 확장하기 쉽습니다.
- 안정성을 우선해 서비스워커 캐싱은 기본 배포본에 넣지 않았습니다. 앱 출시 단계에서만 버전 관리와 캐시 무효화 전략을 함께 설계하는 것이 안전합니다.
- 앱 패키징 시에도 데이터 원본은 동일하게 구글 캘린더의 앱 데이터 이벤트로 유지하는 것을 권장합니다.

## 일정 생성 방식

- 계절별 물주기 보정을 반영하기 위해 무한 반복 일정 대신 앞으로 6개월치 개별 일정을 생성합니다.
- 일정 생성과 삭제는 가능한 경우 구글 배치 요청으로 묶고, 배치 요청이 실패하면 순차 처리로 fallback합니다.
- 앱에서 식물을 수정하거나 `물 줬어요!`를 누르면 기존 미래 일정을 정리한 뒤 다시 생성합니다.
- 상단 `저장`을 실행하면 이 앱이 만든 물주기 알림 일정을 일괄 정리한 뒤, 중앙 데이터 기준으로 다시 생성합니다.
- 앱에 없는 화분 일정은 재생성 과정에서 제거되므로, 화분 삭제와 이름 변경 후 남은 예전 일정을 정리할 때도 같은 기능을 사용합니다.
- 새 알림 일정은 식물 식별 정보를 사용자에게 보이는 설명에 넣지 않고, 구글 캘린더의 비공개 확장 속성으로 식별합니다. 구버전 일정 정리를 위해 예전 메타데이터 검색은 유지합니다.

## 개인정보

이 저장소에는 개인 Google Client ID, API Key, 캘린더 ID가 포함되어 있지 않습니다. 사용자가 앱 화면에서 입력한 API 설정은 해당 브라우저의 `localStorage`에 저장되고, 구글 연동 후 화분 데이터는 사용자가 지정한 구글 캘린더에 저장됩니다.
