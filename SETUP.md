# 물 줘! 설치 및 사용 설명서

## 1. 앱 실행 방법

### 방법 A. 식물 관리만 사용하기

1. 이 저장소를 다운로드합니다.
2. `index.html` 파일을 브라우저로 엽니다.
3. `새 화분 등록`을 눌러 식물을 추가합니다.

이 방식은 구글 로그인이 안정적으로 동작하지 않을 수 있습니다. 캘린더 연동까지 사용하려면 방법 B 또는 방법 C를 사용하세요.

### 방법 B. 로컬 서버로 실행하기

Python이 설치되어 있다면 프로젝트 폴더에서 아래 명령을 실행합니다.

```powershell
py -3 -m http.server 8765 --bind 127.0.0.1
```

브라우저에서 아래 주소로 접속합니다.

```text
http://127.0.0.1:8765/index.html
```

### 방법 C. GitHub Pages로 배포하기

1. GitHub에 새 저장소를 만듭니다.
2. 이 폴더의 파일을 저장소에 업로드합니다.
3. GitHub 저장소의 `Settings` > `Pages`로 이동합니다.
4. `Deploy from a branch`를 선택합니다.
5. 브랜치는 `main`, 폴더는 `/root`를 선택합니다.
6. 저장 후 GitHub Pages 주소로 접속합니다.

주소 예시는 아래와 같습니다.

```text
https://사용자이름.github.io/저장소이름/
```

## 2. 구글 캘린더 연동 준비

구글 캘린더 연동을 사용하려면 Google Cloud Console에서 사용자별 API 설정이 필요합니다.

### 2-1. Google Cloud 프로젝트 만들기

1. Google Cloud Console에 접속합니다.
2. 새 프로젝트를 만듭니다.
3. `API 및 서비스` > `라이브러리`로 이동합니다.
4. `Google Calendar API`를 검색해 사용 설정합니다.

### 2-2. OAuth 동의 화면 설정

1. `API 및 서비스` > `OAuth 동의 화면`으로 이동합니다.
2. 앱 유형은 보통 `외부`를 선택합니다.
3. 앱 이름, 사용자 지원 이메일 등을 입력합니다.
4. 테스트 모드로 사용할 경우 `테스트 사용자`에 앱을 사용할 구글 계정을 모두 추가합니다.

### 2-3. API Key 만들기

1. `API 및 서비스` > `사용자 인증 정보`로 이동합니다.
2. `사용자 인증 정보 만들기` > `API 키`를 선택합니다.
3. 발급된 API Key를 복사합니다.

가능하면 API Key 제한에서 HTTP referrer를 설정하세요.

로컬 서버 예시:

```text
http://127.0.0.1:8765/*
```

GitHub Pages 예시:

```text
https://사용자이름.github.io/*
```

### 2-4. OAuth Client ID 만들기

1. `사용자 인증 정보 만들기` > `OAuth 클라이언트 ID`를 선택합니다.
2. 애플리케이션 유형은 `웹 애플리케이션`을 선택합니다.
3. `승인된 JavaScript 원본`에 앱을 실행할 주소의 원본을 추가합니다.

로컬 서버 예시:

```text
http://127.0.0.1:8765
```

GitHub Pages 예시:

```text
https://사용자이름.github.io
```

주의: `승인된 JavaScript 원본`에는 `/저장소이름/` 같은 경로를 넣지 않고 도메인 원본까지만 입력합니다.

## 3. 식물관리 캘린더 만들기

1. Google Calendar에 접속합니다.
2. 새 캘린더를 만듭니다.
3. 이름은 예를 들어 `물 줘! 식물 관리`로 정합니다.
4. 해당 캘린더 설정으로 이동합니다.
5. `캘린더 통합` 섹션의 `캘린더 ID`를 복사합니다.

이 캘린더를 가족이나 함께 관리할 사람에게 공유할 수 있습니다. 알림을 받을 사람은 각자 자신의 Google Calendar 앱에서 이 공유 캘린더의 알림을 켜는 것을 권장합니다.

## 4. 앱에서 연동하기

1. 앱을 엽니다.
2. `캘린더 연동` 탭으로 이동합니다.
3. 아래 값을 입력합니다.
   - Google Client ID
   - Google API Key
   - 식물관리 캘린더 ID
4. `설정 저장`을 누릅니다.
5. `구글 계정 연동`을 누르고 구글 로그인을 완료합니다.
6. 화분을 등록한 뒤 `구글 캘린더 일정 전체 재생성`을 누릅니다.

`구글 계정 연동`을 완료하면 앱은 현재 등록된 화분 목록을 기준으로 구글 캘린더의 물주기 일정을 정리한 뒤 다시 생성합니다. 예전에 삭제했거나 이름을 바꾸기 전의 잘못된 일정이 남아 있을 때도 이 방식으로 정리됩니다.

## 5. 사용 방법

### 화분 등록

1. `새 화분 등록`을 누릅니다.
2. 식물 종류를 입력합니다.
3. 도감에 있는 식물은 물주기 주기가 자동으로 입력됩니다.
4. 마지막으로 물 준 날짜를 입력합니다.
5. 계절 자동 보정을 사용할지 선택합니다.
6. `등록하기`를 누릅니다.

### 물 줬어요

화분에 물을 준 뒤 카드의 `물 줬어요!`를 누릅니다. 마지막 물준 날짜가 오늘로 바뀌고 다음 일정이 다시 계산됩니다.

### 식물 도감 추가

1. `식물 도감` 탭으로 이동합니다.
2. 식물 이름, 물주기 주기, 빛 환경, 관리 팁을 입력합니다.
3. `도감에 추가`를 누릅니다.

추가한 도감 데이터는 현재 브라우저에 저장됩니다.

## 6. 운영 기준

- 앱이 원본 데이터입니다.
- 구글 캘린더는 알림과 표시를 위한 사본입니다.
- 캘린더에서 일정을 직접 삭제하거나 이름이 꼬이면 `구글 캘린더 일정 전체 재생성`으로 앱 기준 일정을 다시 만듭니다.
- 화분을 완전히 삭제하려면 앱에서 삭제하세요.
- 네이버 캘린더 등 외부 캘린더는 보기용으로 사용하는 것을 권장합니다.
- 알림은 Google Calendar 앱에서 받는 것이 가장 안정적입니다.

## 7. 문제 해결

### 구글 로그인이 되지 않을 때

- `file://`로 열고 있지 않은지 확인합니다.
- Google Cloud Console의 `승인된 JavaScript 원본`에 현재 접속 주소가 등록되어 있는지 확인합니다.
- OAuth 테스트 모드라면 현재 로그인 계정이 테스트 사용자에 포함되어 있는지 확인합니다.

### 캘린더에 일정이 생기지 않을 때

- 캘린더 ID가 정확한지 확인합니다.
- 로그인한 구글 계정이 해당 캘린더를 수정할 권한이 있는지 확인합니다.
- 앱에서 `구글 캘린더 일정 전체 재생성`을 다시 누릅니다.

### 삭제한 화분 일정이 캘린더에 남아 있을 때

- 앱에서 구글 계정을 다시 연동하거나 `구글 캘린더 일정 전체 재생성`을 누릅니다.
- 이 앱이 만든 물주기 일정만 정리한 뒤 현재 앱에 살아있는 화분 일정만 다시 생성합니다.
- 그래도 남는 일정이 있다면 예전 버전에서 만든 일정일 수 있으니 캘린더 ID가 식물관리 전용 캘린더인지 확인하세요.

### 알림이 오지 않을 때

- Google Calendar 앱에서 해당 캘린더 표시와 동기화가 켜져 있는지 확인합니다.
- Google Calendar 웹 설정에서 해당 캘린더의 이벤트 알림을 설정합니다.
- 네이버 캘린더에 구글 캘린더를 연동한 경우 알림까지 안정적으로 전달되지 않을 수 있습니다.
