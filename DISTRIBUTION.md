# 💧물 줘! 배포 안내

이 폴더는 식물 물주기 일정 관리 앱 `💧물 줘!`의 배포용 파일입니다. 개인 Google API 정보나 캘린더 ID는 포함되어 있지 않으므로, 사용하는 사람이 각자 자신의 구글 계정으로 설정해야 합니다.

## 포함 파일

- `index.html`: 앱 실행 파일
- `style.css`: 화면 디자인
- `app.js`: 앱 동작 로직
- `plants-db.js`: 기본 식물 도감 300종 데이터
- `README.md`: 앱 소개
- `SETUP.md`: 설치와 구글 캘린더 연동 설명서
- `PROJECT_NOTES.md`: 프로젝트 운영 메모

## 가장 쉬운 사용 방법

캘린더 연동 없이 식물 목록만 관리하려면 `index.html`을 브라우저에서 열면 됩니다. 이 경우 데이터는 현재 기기에만 저장됩니다.

PC와 모바일에서 같은 데이터를 쓰거나 구글 캘린더 알림까지 쓰려면 `SETUP.md`의 설명대로 로컬 서버나 GitHub Pages 주소에서 실행해야 합니다. `file://`로 직접 열면 구글 로그인이 제한될 수 있습니다.

## 캘린더 운영 원칙

- 구글 캘린더 안의 앱 데이터 이벤트가 원본입니다.
- 물주기 알림 일정은 중앙 데이터에서 만들어지는 사본입니다.
- 화분 추가, 수정, 삭제는 앱에서 합니다.
- 처음 한 번은 톱니바퀴 설정에서 Google Client ID, API Key, 캘린더 ID를 저장합니다.
- 이후에는 상단 `구글 로그인`만 누르면 중앙 데이터를 불러옵니다.
- 캘린더 일정이 꼬였거나 예전 화분 일정이 남아 있으면 앱 상단의 `저장`을 누릅니다.
- 이 기능은 이 앱이 만든 물주기 알림 일정을 정리한 뒤, 중앙 데이터에 살아있는 화분 기준으로 다시 만듭니다.
- 저장과 동기화 작업은 앱 내부에서 순서대로 처리되며, 구글 토큰 만료 시 1회 자동 재시도합니다.
- 일정 삭제와 생성은 가능한 경우 배치 처리하고, 실패 시 순차 처리로 전환합니다.
- `💧물 줘! 앱 데이터 (삭제 금지)` 일정은 PC/모바일 공유를 위한 중앙 데이터이므로 지우지 않습니다.

## 알림 권장 방식

알림은 Google Calendar 앱에서 받는 방식이 가장 안정적입니다. 네이버 캘린더 같은 외부 캘린더에 구글 캘린더를 연결하면 일정 확인은 가능하지만, 알림까지 안정적으로 전달되지 않을 수 있습니다.

## 개인정보

이 배포본에는 개인 API Key, OAuth Client ID, 캘린더 ID, 식물 관리 데이터가 들어 있지 않습니다. 구글 연동 전 임시 데이터와 API 설정은 사용하는 사람의 브라우저 `localStorage`에 저장되고, 구글 연동 후 화분 데이터는 지정한 구글 캘린더에 저장됩니다.
